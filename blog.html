1. Difference between Map, Filter, ForEach & Find ?
ForEach: The forEach method allows us to run a function for every element of the array. ForEach doesn’t return anything.
Map: The map method is very similar to the forEach method—it allows us to execute a function for each element of an array. But the difference is that the map method creates a new array using the return values of this function.
Find: The find function returns the first item that matches the callback function, or undefined if there is nothing that matches.
Filter: Filter works like the find function, but instead of returning the first matched item, filter will return the array of matched items or an empty array if nothing matched.
2. Difference between Local Storage, Session Storage & Cookies ?
Local Storage: This read-only interface property provides access to the Document’s local storage object, the stored data is stored across browser sessions. Similar to sessionStorage, except that sessionStorage data gets cleared when the page is closed.
Session Storage: Session Storage objects can be accessed using the sessionStorage read-only property. The difference between sessionStorage and localStorage is that localStorage data does not expire, whereas sessionStorage data is cleared when the page session ends.
Cookies: Cookies are data, stored in small text files as name-value pairs, on our computer. New cookies can also be created via JavaScript using the Document.cookie property, and if the HttpOnly flag is not set, existing cookies can be accessed from JavaScript as well.
3. What is Event Loop? How it's works ?
Event Loop: JavaScript has a runtime model based on an event loop, which is responsible for executing the code, collecting and processing events, and executing queued sub-tasks. The Event Loop has one simple job — to monitor the Call Stack and the Callback Queue. If the Call Stack is empty, the Event Loop will take the first event from the queue and will push it to the Call Stack, which effectively runs it.
How it's works: The event loop runs continuously and checks for any pending tasks in a queue. If a task is found, the event loop moves it to the execution stack for processing. If the execution stack is empty, the event loop waits until a task is added to the queue. Once a task is executed, any associated callback functions are added to the event queue. These callbacks are executed in the order in which they were added, and the event loop continues to process tasks and callbacks in this way.
